# Pizza Doc v0.1.0

*First public release.*

## Highlights

- **File-based architecture-as-code.** Describe your system as YAML —
  actors, modules, components, models, tables, use cases — and Pizza
  Doc does the rest.
- **Strict three-pass validator.** 25+ rules across schema, reference
  resolution, and semantic checks. `REF_BROKEN` suggestions via
  Levenshtein. Warnings vs errors calibrated by real-world noise on
  the demo space.
- **Local-first web UI.** Vite + React 18. File System Access API —
  your files never leave the browser. Dark mode, ⌘K command palette,
  keyboard-first everywhere.
- **Interactive sequence canvas.** Four levels — `L1 Tiers · L2 Modules
  · L3 Components · L4 Methods` — switchable via the level pill strip,
  with tier colours (frontend / backend / database / external / queue)
  carried through participant chips and lifelines. Pan to drag,
  ⌘/ctrl + scroll to zoom (clamped 0.3×–3×), dotted grid scales with
  zoom. Click-to-inspect, number-keys to jump to step N. Hover any
  arrow carrying a `via: <DTO>` for a popover with the first five
  fields.
- **Space landing = primary use case.** `/space/$spaceId` renders the
  richest use-case canvas inline (most-steps wins, ties broken by id);
  spaces with zero use cases fall back to the stats card.
- **Undo/redo + autosave.** The Zustand store snapshots the whole space
  per edit, capped at 50 entries. File watcher picks up external disk
  edits within 2 seconds.
- **AI export.** `exportSpaceForAi()` emits a single flat Markdown file
  with angle-bracketed refs, designed for LLM contexts. Accessible via
  the top-bar ⋯ menu or the ⌘K palette.
- **ZIP export.** Whole-space download via fflate.

## What's in the box

- `@pizza-doc/core` — schemas, loader, validator, serializer, AI
  exporter, ref index. Pure TypeScript, no FS deps in the main entry.
- `@pizza-doc/cli` — public npm binary package for scaffolding,
  validation, reporting, drift checks, and export.
- `@pizza-doc/web` — the UI. Vite + React 18 + TanStack Router +
  Zustand + shadcn/ui + React Flow.
- `spaces/pizza-shop-demo/` — a seven-use-case demo space covering 3
  actors, 4 modules (frontend, service, DB, external), 7 use cases
  including a cross-module `place-order` flow.
- Docs site under `docs/site/` — Astro Starlight. Getting started, four
  concept pages, four reference pages.

## Known limitations

- **CLI surface is young.** The public binary is useful for validation,
  scaffolding, drift checks, and export, but command names and report
  formatting can still change before v1.
- **Schema version story is still conservative.** v0.2 contract fields are
  supported in code paths that need them, but docs and migration guidance
  still need a dedicated pass before calling the project stable.
- **Browser support.** File System Access API only — Chrome, Edge,
  Brave, Arc, Opera. No Firefox or Safari until we ship the Tauri
  wrapper.
- **No ignore directives.** `# pizza-doc-ignore: RULE_CODE` pragmas
  aren't supported and aren't planned for v0.1 — we'd rather tune rule
  fidelity than paper over it.
- **No multi-space comparison.** The UI loads one space at a time.

See [`docs/backlog.md`](https://github.com/pizza-doc/pizza-doc/blob/main/docs/backlog.md)
for the full roadmap.

## Upgrade path

This is the first release; there's nothing to upgrade from.

The schema declares `pizzaDocVersion: 0.1.0`. v0.2 will bump it to
`0.2.0` and add the additive fields described above — no migration
script should be required for v0.1 spaces.

## Thanks

Every rule in the semantic validator was stress-tested against the
pizza-shop demo. Every keyboard chord in `SpaceLayout.tsx` was tuned
against real editing sessions. The bar for v0.1 was "could I use this
on my own codebase tomorrow" — it clears the bar.

```bash
cd pizza-doc
pnpm install
pnpm build
pnpm --filter @pizza-doc/web dev
```
